## 🚀 Day 1: AWS Fundamentals & Networking

### 🔹 Why AWS?
- Scalable, reliable, and cost-effective cloud platform
- Global infrastructure with high availability
- Wide range of services

### 🌍 Regions & Availability Zones
- Regions = geographic locations
- AZs = isolated data centers within regions
- Designed for fault tolerance and high availability

### 🌐 VPC (Virtual Private Cloud)
- Created a custom VPC

#### Key Components:
- CIDR Block (e.g., 10.0.0.0/16)
- Subnets (public/private)
- Route Tables (traffic control)
- Internet Gateway (internet access)
- Security Groups (firewall rules)

📌 **Outcome:**  
Understood how AWS networking is structured and how secure environments are built.
